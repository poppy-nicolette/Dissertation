Title: Babel での Babble の分類: OpenAlex データベース上の言語検出アルゴリズムのパフォーマンスの評価 Sorting the Babble in Babel: Assessing the Performance of Language Detection Algorithms on the OpenAlex Database
Abstract: このプロジェクトは、さまざまな言語分類手順、さまざまな Python 言語検出アルゴリズムを組み合わせた手順、および OpenAlex データベースからサンプリングされた手動で注釈が付けられた記事から抽出されたメタデータ ベースのコーパスを比較することを目的としています。各アルゴリズム、コーパス、言語の精度と再現率のパフォーマンス、および各アルゴリズムとコーパス タイプで記録された処理速度の分析に続いて、各アルゴリズム、コーパス、言語の確率混同行列と、OpenAlex データベース全体の相対記事言語頻度の確率モデルを使用して、データベース レベルでの全体的な手順のパフォーマンスがシミュレートされました。結果は、プロシージャのパフォーマンスが、実装された各対策に与えられた重要性に大きく依存することを示しています。精度が優先されるコンテキストでは、貪欲コーパスで LangID アルゴリズムを使用すると最良の結果が得られます。ただし、再現率が精度よりも少なくともわずかに重要であると考えられるすべてのケース、または処理時間に何らかの考慮が与えられるとすぐに、FastSpell アルゴリズムとタイトル コーパスを組み合わせた手順は、他のすべての方法よりも優れています。真に多言語対応の大規模書誌データベースが存在しないことを考えると、これらの結果が、言語を超えた書誌ベースの研究と分析に対する OpenAlex データベースの比類のない可能性を確認し、促進するのに役立つことが期待されます。 This project aims to compare various language classification procedures, procedures combining various Python language detection algorithms and metadata-based corpora extracted from manually-annotated articles sampled from the OpenAlex database. Following an analysis of precision and recall performance for each algorithm, corpus, and language as well as of processing speeds recorded for each algorithm and corpus type, overall procedure performance at the database level was simulated using probabilistic confusion matrices for each algorithm, corpus, and language as well as a probabilistic model of relative article language frequencies for the whole OpenAlex database. Results show that procedure performance strongly depends on the importance given to each of the measures implemented: for contexts where precision is preferred, using the LangID algorithm on the greedy corpus gives the best results; however, for all cases where recall is considered at least slightly more important than precision or as soon as processing times are given any kind of consideration, the procedure combining the FastSpell algorithm and the Titles corpus outperforms all other alternatives. Given the lack of truly multilingual, large-scale bibliographic databases, it is hoped that these results help confirm and foster the unparalleled potential of the OpenAlex database for cross-linguistic, bibliometric-based research and analysis.
